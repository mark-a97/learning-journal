Overall Summary

Published on December 04, 2018

I learnt to:

Overall Summary In the first semester of university I feel like I have learned a lot in web development, but also still have more progress to make. I started with small amounts of experience with CSS and HTML, but now I have widened that knowledge and know a lot more. From creating lists, tables, and buttons, over to implementing grids, contact forms and responsive web pages.

My most successful week I felt was creating the grid. Although I struggled because I didn’t understand it properly and struggled to make it work, when I finally understood and sorted the problems it felt like a big achievement. From this, I was then able to add the grid to another one of my pages and change it to how I wanted it to be.

My most unsuccessful week was week 7 where I implemented flexible media into my web pages. Although it wasn’t that difficult, I had a few issues regarding the banner. Sometimes the banner was too big, or the margins weren’t correctly positioned and when the screen was made smaller the banner wouldn’t adjust. This took some time to fix, but as with everything it is always fixable if you put your mind to it and keep working on it. Another smaller issue I had here was converting the CSS I had in my HTML over to my CSS document.

I still have much to learn regarding creating web pages, styling them, and in the design department of things. I have used some examples from other websites and referenced them in my journal as some parts of code such as the ‘Go to top’ button was a bit advanced for me to do off the top of my head, without any previous knowledge of this type of styling. I look forward to carrying on this progress

I have also added some javascript to my web page. This is a change from the original button as it wasn't being as responsive as I wanted it to. I got this information from an online source which is referenced.

Week 9 - HTML and CSS Forms

Published on November 27, 2018

I learnt to:

  1. Insert a form into a web page
  2. Use different input types
  3. The use of fieldsets and legends
  4. How to include form validation
Week 9 Summary Week 9 I found particularly interesting apart from struggling with the slider which would just not seem to work. I believe now that to make it work I would need to include some JavaScript. The start of the form was just using the simple id attribute. Then it started to get interesting by adding the form elements such as labels, radio buttons, fieldsets etc.

When adding the radio buttons, I wanted them to look slightly more modern and better looking, so I had looked up how to add radio buttons in my CSS. I have introduced code from an example and changed parts to fit on my contact page’s form.

What I did find interesting this week, was the fieldset and legend elements. I do like the look of them and how they work so I have included them in my summaries throughout the learning journal. This is the same with the validation, which of course when making a form you are going to need. Using the ‘required’ attribute will stop users from missing out important fields of a form.

I found the CSS styling quite easy compared to other weeks. It was what I have done before, just changing border styles, colours, widths etc. Not anything that required a struggle.

Week 8 - CSS3 Responsive Layout

Published on November 20, 2018

I learnt to:

  1. Use flexbox
  2. Use multi-columns
  3. Use more than one stylesheet
Week 8 Summary In week 8 I changed a lot on my web pages. At first, all the pages were all designed very similarly when you looked at them. They had the exact same grid, layout, font, headings etc. Consistency is good, but not when making step by step instructions. Here the card interface was introduced, along with a new stylesheet.

First of all, when I was trying to create the cards, I was still using the original stylesheet. This made this very difficult for me as it was not working as it was supposed to. After making a new stylesheet and linking it to the tutorial page, it started to come around and look a lot neater. I had also been typing up the cards wrong in the html, which was causing them to appear wrong. Each time I were putting a new div called “instructions” after each card. This was putting the cards in a singular column, rather than setting them out how they appear now.

That was all that really went wrong this week, and it took me a little while to figure out such a small mistake. But what went well was the styling of the cards, I found it very easy going back over the CSS and making sure I understood what was going on.

Week 7 - Flexible Media

Published on November 13, 2018

I learnt to:

  1. Insert flexible images
  2. Inserting flexible images with text wrapping
  3. Using a flexible video container
  4. Adding a banner into my web page
  5. Using background images
Week 7 Summary In week 7 there were a lot of changes that happened to my web page. This was mainly due to having it reviewed by the class. There were a few issues with my web page such as margins, padding, font types, styles, and text alignment. All these issues I quickly addressed, and it started to make my web page look better already.

There was also no way to tell which page I was on (learning journal, tutorial or contact me page). To fix this I looked up buttons and added in a box shadow, now it is clearer to the user to see what page they are on. My navigation bar was also not how I wanted it to be, I wanted the links to be separated across the page, rather than being clumped together. To fix this I used an element called “justify-content: space-between”. This separated out my navigation bar and made it look exactly how it wanted to look.

Other than the review where my work was checked, during this week I a banner to one of my web pages (the tutorial page) and a background image to the rest. I didn’t add it to the learning journal or contact page because I didn’t see it necessary. I also added a flexible image which is kept at the very top of my web page, with some writing around it. I didn’t see a need to include an image in my journal so I have kept it there.

I also found it bad practice to put CSS into my html files, so instead of doing that I converted it to my stylesheets.

I learnt to:

  1. Create space between text using CSS
  2. Use a dropcap class to give my learning journal a article look
  3. Make my text work around an image instead of having it above or underneeth/above
  4. Use 'float' to make my text either go to the left or right of the screen/image

If I were to write an article I would use one in my web pages. Week 5 for me I was starting to get used to CSS again, I learned how to enable text to work around the image rather than just sit above or below it. This made it more professional and gave it a more stylish look.

I am happy at the stage and progress of my website so far, I feel like everything is going well and I am progressing at a good rate. Along every week I have been checking both my html and CSS in the W3C validator, I have had some minor issues along the way but none that are difficult to fix.

Week 4 - CSS basics

Published on October 23, 2018

I learnt to:

  1. Add and link css to my html
  2. Change the font, size and colour of specific parts using css
  3. Style other elements such as headings, tables, lists etc.
  4. Style the default page elements by editing the #wrapper in css
  5. Use the class selector. Here I have used it on multiple occasions to seperate different parts of the site
  6. Use and edit buttons, now I can use a bigger variety
  7. Use CSS more rather than having it in the html. In this case I removed the table from my HTML and put it in my CSS
Week 4 Summary Week 4 for me was again more of a refresher week as I had not done any CSS for quite some time. I could recognise what was happening and understand the elements being used but was just not up to date. Therefore, making small changes to parts like the font and colours were very helpful. Shortly after getting used to this I’d go over the classes and understand what was being changed. In college, I was always told it was wrong or a bad habit to include CSS in your html, so where applicable I changed my html so that it was strictly html in there.

I had used buttons before but never went through the different effects, so I looked this up and changed the navigation bar buttons and the button where it sends you to the top of the page. With my table, I felt like it looked a little untidy having the same colour throughout, so I changed it to where if part of the table is empty, it will change to a different colour. Although I said previously about having some CSS in my html there were some parts that I left out, so when I validated it I had a couple of errors. It was another simple fix where I just needed to change both my html and CSS.
